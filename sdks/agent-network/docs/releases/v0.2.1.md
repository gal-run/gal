# Agent Network v0.2.1

Private dependency hardening release.

## Packaging Changes

- `@gal/agent-network` no longer installs `@gal/agents` from the public
  `gal-run/gal-agents` Git URL.
- The pinned `@gal/agents@0.1.0` contract artifact is vendored at
  `vendor/gal-agents-v0.1.0` and bundled into the `@gal/agent-network` package
  artifact.
- CI no longer configures public GitHub URL rewrites for the agent definition
  dependency.

## Boundary

`@gal/agents` remains the canonical source of truth for Agent Card, task,
identity, scope, audit, artifact, health, and status definitions. The vendored
artifact is a pinned distribution copy used so `gal-run/gal-agents` can remain
private while `@gal/agent-network` is still self-contained during the v0.x
migration window.
