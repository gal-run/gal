# Agent Network v0.2.0

Boundary split release.

## Contract Changes

- `@gal/agents` is now the canonical source of truth for GAL Agent Card, task,
  identity, scope, audit, artifact, health, and status definitions.
- `@gal/agent-network` depends on `@gal/agents@v0.1.0`.
- Historical `GalService*` exports remain as compatibility aliases during the
  v0.x migration window.
- SDK task creation now uses `agentId` and `calleeAgentId` semantics internally.

## Boundary

`@gal/agents` defines what an agent is. `@gal/agent-network` defines how agents
discover, route, delegate, and interoperate across transports such as A2A, MCP,
HTTP/JSON, gRPC, GraphQL, and CLI.
