# v0.5.0

Adds token-throughput capacity forecasting and running cluster utilization decisions.

## Included

- model throughput profiles with provider, GPU type/count, context window, concurrency, token throughput, lifecycle timing, minimum billing, and hourly cost
- per-task token capacity forecasts for prefill, decode, reasoning, runtime, and context fit
- aggregate cluster-capacity forecasts for planned minutes, billable minutes, token utilization, and projected cost
- runtime cluster snapshot and utilization contracts
- `calculateGalPredictionClusterUtilization` for `scale_up`, `hold`, `drain`, and `shutdown` decisions while the swarm is running
- README guidance for RunPod GraphQL runtime metrics, Crusoe VM metrics, and in-cluster model queue/token metrics
